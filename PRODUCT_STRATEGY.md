# AI Guide: Product Strategy Document

**Last Updated:** March 2026
**Document Owner:** Product Team
**Status:** Strategic Planning Phase

---

## Executive Summary

AI Guide is transforming from an educational platform teaching Claude Code and VS Code basics into a **collaborative AI knowledge-sharing ecosystem** where developers of all levels can share workflows, contribute to a community knowledge base, verify skills, and discover trending AI development practices.

This shift addresses a critical gap: there's no centralized, community-driven platform for sharing Claude Code expertise, development workflows, and AI integration patterns. While platforms like dev.to exist for general software content and GitHub Discussions for project-specific support, there's no dedicated space optimized for Claude Code workflows, AI agent configuration, and AI-augmented development practices.

**Core Thesis:** By enabling developers to share their Claude Code knowledge, workflows, and verified skills, we create network effects where each contribution increases platform value for all users—turning the platform into the canonical reference for AI-augmented development.

## Competitive Analysis

### Direct Competitors

#### 1. **Cursor Community / Cursor Discussions**
- **Strengths:** Integrated with popular IDE, active user base, focused community
- **Weaknesses:** IDE-specific, no knowledge base structure, limited discoverability
- **Opportunity:** AI Guide is tool-agnostic (Claude Code, VS Code, etc.) and focused specifically on Claude

#### 2. **GitHub Discussions**
- **Strengths:** Already where developers are, integrated with repositories, GitHub reputation
- **Weaknesses:** Repository-focused, scattered across repos, no curation, hard to find patterns
- **Opportunity:** Centralized, curated, searchable knowledge base; skill verification feature

#### 3. **Stack Overflow**
- **Strengths:** Trusted source, massive reach, high Google ranking
- **Weaknesses:** Q&A format poor for workflows, no skill verification, slow to adapt to AI-specific content
- **Opportunity:** Workflow and case study format better suited than Q&A; real-time trending; skill registry

#### 4. **dev.to**
- **Strengths:** Creator-friendly, good discoverability, strong community
- **Weaknesses:** General software platform (not AI-focused), limited skill verification, no workflow templates
- **Opportunity:** Specialized platform for Claude Code + AI workflows; better UX for workflows; skill verification

#### 5. **Hashnode / Medium**
- **Strengths:** Blogging platform, good writing tools, creator monetization
- **Weaknesses:** Long-form content focus, no skill registry, no collaborative workflow features
- **Opportunity:** Workflow-focused format, skill verification, trending AI content API integration

### Competitive Positioning

**AI Guide is the only platform that:**
1. Specializes exclusively in Claude Code and Claude API workflows
2. Includes interactive skill verification tools
3. Integrates X (Twitter) API to surface trending AI development discussions
4. Enables collaborative, updatable knowledge base (wiki-style) for workflows
5. Bridges the gap between beginner education and expert knowledge sharing
6. Focuses on development process documentation (not just final code)

**Market Positioning:**
- **Not a coding tutorial site** (like Codecademy)
- **Not a question-answer forum** (like Stack Overflow)
- **Not a general blogging platform** (like Medium)
- **Is a specialized community knowledge platform** for Claude Code workflows with skill verification and trend discovery

## Key Metrics to Track

### North Star Metric
**Weekly Active Workflow Consumers:** Users who consume at least one workflow, skill verification, or trending article per week

*Why:* Measures ongoing platform engagement; includes both content creators and consumers; indicates product-market fit

### Primary Engagement Metrics

1. **Daily Active Users (DAU)**
   - Target: 1K by week 4, 5K by week 12, 50K by month 6
   - Indicates sustained engagement
   - Segment by: contributor vs. consumer, new vs. returning

2. **Weekly Active Users (WAU)**
   - Target: 3x DAU (engagement goal)
   - Better metric than DAU for weekly products

3. **Monthly Active Users (MAU)**
   - Target: 10K by month 3, 50K by month 6
   - Platform health indicator

4. **Content Creation Rate**
   - Target: 10 new articles/workflows per day by week 12
   - Measure: Articles published per 1,000 MAU
   - Indicates healthy creator ecosystem

5. **Comment/Discussion Rate**
   - Target: 2+ comments per article on average
   - Indicates community engagement beyond passive consumption
   - Track: Comments per article, discussion threads

### Community Health Metrics

1. **Contributor Activation Rate**
   - % of new users who publish content within 30 days
   - Target: 5-8%
   - Indicator of platform ease of use and value proposition clarity

2. **Contributor Retention**
   - % of contributors who publish 2+ items
   - Target: 40%+ at 8 weeks
   - Indicates sustainable creator base

3. **Skill Verification Adoption**
   - # of users who completed skill verification challenge
   - Target: 500+ by week 14
   - Measure: % of content creators with verified skills (target 30%)

4. **Community Quality Score**
   - Rating system: content quality, comment quality, helpfulness
   - Target: 4.0+ average rating
   - Manual spot-checks + sentiment analysis

### Content Performance Metrics

1. **Average Content Lifespan**
   - Measure: Views over time (decay rate)
   - Target: 50% of views within first 7 days, but evergreen content should have long tail
   - Indicates quality and searchability

2. **Search Discovery Rate**
   - % of views coming from search vs. feed
   - Target: 60%+ by month 6
   - Indicates SEO health and content relevance

3. **Trending Integration Engagement**
   - Click-through rate on trending X posts
   - Target: 5-10% CTR
   - Conversion: trending post viewer → registered user (target 2%)

4. **Collaborative Edit Rate**
   - % of articles with edits from non-original-authors
   - Target: 20%+ by month 4
   - Indicates wiki mode adoption

### Business Metrics

1. **Time to First Contribution**
   - Avg. days from signup to first published content
   - Target: < 7 days
   - Benchmark against similar communities (dev.to = 14 days avg)

2. **Content Upload Time**
   - Average time to publish an article from starting draft
   - Target: < 30 minutes
   - Indicates friction in content creation UX

3. **Virality Coefficient**
   - How many new users are brought by existing users
   - Target: 1.2+ (each user brings 0.2 new users through sharing/referral)
   - Key to viral growth

4. **Unit Economics (if monetized)**
   - CAC: Cost to acquire each user
   - LTV: Lifetime value of user
   - Target LTV:CAC > 3:1
   - Creator LTV: Value generated by top creators

### Trending/Discovery Metrics

1. **Trending Feed Engagement**
   - # of users visiting trending section weekly
   - Target: 40%+ of WAU
   - Clicks on trending posts
   - Measure: 10% of traffic from trending section by month 3

2. **Topic Diversity**
   - # of unique skills/topics covered
   - Target: 50+ core skills by month 6
   - Ensures platform serving broad audience

3. **Freshness Score**
   - Measure: avg. age of top 10 articles
   - Target: < 30 days
   - Indicates active, current knowledge base

### Retention & Loyalty Metrics

1. **Day-1, Day-7, Day-30 Retention**
   - % of users returning after 1, 7, 30 days
   - Target: 60% day-1, 40% day-7, 25% day-30
   - Baseline for healthy engagement

2. **Net Promoter Score (NPS)**
   - Quarterly survey: "How likely to recommend?"
   - Target: 40+ by month 3, 60+ by month 6
   - Benchmark: dev.to (65), Stack Overflow (45)

3. **Churn Rate**
   - % of users inactive for 30+ days
   - Target: < 5% monthly churn for active base
   - Focus on preventing contributor churn (target < 2%)

## Risk Assessment

### High-Risk Items

#### Risk 1: Low Content Contribution Rate
**Probability:** High | **Impact:** Critical
- Most platforms suffer from "1% creators, 99% lurkers"
- Threshold: If < 2% of users create content, network effects won't materialize

**Mitigation:**
- Aggressive onboarding for first contribution
- Suggest templates for common workflow types
- Create incentives early (badges, recognition, revenue share)
- Feature new creator content prominently
- Reach out personally to high-value contributors
- Start with seed content from expert practitioners

**Monitor:** Track contributor activation rate weekly; trigger action if < 3%

---

#### Risk 2: Low Content Quality / Platform Spam
**Probability:** Medium | **Impact:** High
- As platform grows, quality control becomes harder
- Spam/low-quality content drives away serious users

**Mitigation:**
- Implement community moderation early (flag, report system)
- Manual curation of featured content from day 1
- Reputation system: low-rated content becomes less visible
- Skill verification creates trust signal (prioritize in feed)
- Clear community guidelines and enforcement
- Automated spam detection (keyword filtering, duplicate detection)

**Monitor:** Content quality score; community voting patterns; feedback sentiment

---

#### Risk 3: Competitive Response from Large Platforms
**Probability:** Medium | **Impact:** High
- Stack Overflow, dev.to, or GitHub could add Claude Code features
- Anthropic could build competing knowledge platform

**Mitigation:**
- Build defensible moat: community + skill verification + trending
- Deep integration with Claude Code ecosystem (VS Code extension)
- Focus on niche, let large platforms follow
- Establish thought leadership and brand affinity
- If Anthropic shows interest: position as partner, not competitor

**Monitor:** Competitive feature launches; Anthropic announcements; developer sentiment

---

#### Risk 4: Dependency on X (Twitter) API
**Probability:** Medium | **Impact:** Medium
- Twitter API changes, pricing increases, or platform decline
- Trending feature is key differentiator

**Mitigation:**
- Add fallback data sources (GitHub trending, HN, Reddit)
- Build trending detector on platform itself (internal posts)
- Have alternative content discovery mechanisms
- Don't make entire monetization depend on trending

**Monitor:** X API status; alternative source viability; usage patterns

---

#### Risk 5: Retention Issues / Virality Not Materializing
**Probability:** Medium | **Impact:** Critical
- Product-market fit might not exist
- Users consume content but don't feel compelled to contribute

**Mitigation:**
- Measure PMF signals from week 1: retention curves, NPS, engagement
- Pivot quickly if signals are weak (A/B test positioning, features, UX)
- Run user interviews: why do people leave? what would make them stay?
- Focus on top users first (Pareto principle: 20% of users drive 80% of value)
- Build referral loop: contributors incentivized to bring peers

**Monitor:** Cohort retention curves; NPS trending; churn reasons (exit surveys)

---

#### Risk 6: Skill Verification Gaming / Credential Inflation
**Probability:** Medium | **Impact:** Medium
- Users might claim skills without genuine proficiency
- Credentials lose value if verification is too easy
- Risk: Users feel deceived by unqualified "verified" contributors

**Mitigation:**
- Design challenges to be genuinely difficult for claimed skill level
- Use randomized, dynamic questions (prevent memorization)
- Start with manual review for verified status
- Track success rate and adjust difficulty
- Implement re-certification requirement (annual renewal)
- Community feedback on verified users (comment "this person really knows this")

**Monitor:** Challenge pass rates; community feedback on verified contributors; credential value perception

---

### Medium-Risk Items

#### Risk 7: Scaling Infrastructure Costs
**Probability:** Medium | **Impact:** Medium
- Video hosting, API calls, database growth expensive
- X API rate limiting could require expensive tier

**Mitigation:**
- Start with lean infrastructure (Vercel, Supabase)
- CDN for content delivery
- X API cost optimization (cache trending results, stagger requests)
- Plan for scaling before hitting limits

---

#### Risk 8: International Language Support Complexity
**Probability:** Medium | **Impact:** Low
- Supporting ko/en/ja adds maintenance burden
- Moderation becomes harder (need native speakers)
- Content might not translate well (code examples, idioms)

**Mitigation:**
- Start with English MVP, add languages in Phase 1
- Community volunteers for translation
- Tooling for automated translation + community review
- Separate moderation teams per language
- Clear content guidelines that travel across languages

---

#### Risk 9: Burnout / Founder Bandwidth
**Probability:** High | **Impact:** Medium
- Building and maintaining community platform is demanding
- Moderation, content curation, user support all time-intensive

**Mitigation:**
- Hire community manager by month 4
- Implement automation: moderation, notifications, recommendations
- Build moderator community early
- Set clear working hours and culture (don't glorify hustle)
- Delegate ruthlessly

---

### Low-Risk Items

#### Risk 10: Technical Debt
**Probability:** Low | **Impact:** Medium
- Moving fast might introduce code quality issues
- Refactoring becomes harder as codebase grows

**Mitigation:**
- Invest in solid foundation (use Next.js patterns, TypeScript, testing)
- Code review process from day 1
- Regular refactoring sprints (10% time)
