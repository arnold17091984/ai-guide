## MVP = Get One Thing Right

**North Star:** Weekly Active Workflow Consumers (WAU)

**Formula for Success:**
1. Get 1,000 users by week 8
2. Get 100+ articles published
3. Hit 40%+ day-7 retention
4. Reach NPS 40+

**If any of these miss badly:** Pivot immediately. Don't push forward.

---

## The Four Pillars (In Priority Order)

1. **Knowledge Sharing** (must work or platform is nothing)
   → Easy publishing, discoverable content, good UX

2. **Skill Verification** (differentiator #1)
   → Interactive challenges, badges, credibility

3. **Trending Discovery** (differentiator #2)
   → X API integration, real-time updates, drives engagement

4. **Community Features** (keeps people engaged)
   → Comments, voting, profiles, discussions

## What NOT to Build (Phase 0)

### Hard No's
- Don't build Claude Code competitor (not your job)
- Don't build IDE (let them stay in VS Code/Cursor)
- Don't build code execution (link to existing sandboxes)
- Don't add NFTs, blockchain, or crypto mechanics
- Don't build AI chatbot as content generator

### "Later" Features
- Wiki-style collaborative editing (Phase 1.5)
- Skill verification challenges (Phase 1.5)
- Monetization (Phase 1.5)
- IDE extensions (Phase 2)
- Advanced recommendations (Phase 2)
- Team workspaces (Phase 1.5)

**Why?** Focus beats features. Master the core first.

## Red Flags (Stop & Regroup)

### Critical Issues (Act Immediately)
| Signal | Action |
|--------|--------|
| < 25% day-7 retention | Audit UX; fix friction points |
| < 1 article published/day by W4 | Onboarding is broken; fix it |
| < 3 comments per 100 views | Commenting UX issue or no engagement |
| NPS < 30 after W4 | Something fundamentally broken |
| > 20% of articles are spam | Moderation needed; implement filters |

### Yellow Flags (Monitor Closely)
- Contributor activation < 3%
- Search views declining
- Trending CTR < 3%
- Churn rate > 15%/week

## Decision Framework

**Before adding a feature, ask:**

1. **Does it move the North Star?** (WAU)
   - Yes → Consider it
   - No → Don't build it

2. **Does it solve a real pain point?**
   - Yes → Validate with users
   - No → Don't build it

3. **Can we build it in 1 week?**
   - Yes → Consider building in MVP
   - No → Defer to Phase 1+

4. **Does it create defensibility?**
   - Yes → Prioritize higher
   - No → Lower priority

5. **Are users asking for it?**
   - Yes → Do it sooner
   - No → Do it later

**Apply to every feature request.**

## When to Pivot

**Keep going if:**
- Users are creating content organically
- Retention is on track
- NPS trending up
- Competitive advantage is clear
- User feedback is positive

**Pivot if:**
- Retention collapses
- No one's publishing content
- Users don't understand value
- Killer competitor launches feature
- Different user segment shows more demand

**Don't wait for perfection. Iterate fast.**
