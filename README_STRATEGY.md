## The Opportunity (TL;DR)

**Problem:** No platform specializes in Claude Code workflows with skill verification and trending discovery.

**Solution:** Community platform where developers share workflows, verify skills, and discover real-time AI trends.

**Why it matters:** 27M developers × 40% using AI tools × growing Claude adoption = massive market with no competitor.

**Why it will win:** 4 differentiators (specialized + verified + trending + community) that are hard to copy together.

**Timeline:** 8 weeks MVP → 6 months Phase 1 → 12 months market leadership.

## Key Strategic Decisions

### 1. **Focus on One Thing: Knowledge Sharing**
- Easy article/workflow publishing
- Discoverable content (search + trending)
- Community feedback (comments + voting)
- Ship this first, perfect it forever

### 2. **Differentiate with Skill Verification**
- Interactive challenges (Phase 1.5)
- Badge system (verified practitioners)
- Trust signal for content creators
- Competitive advantage vs. all competitors

### 3. **Own Real-Time Trends**
- X (Twitter) API integration
- Surface trending Claude Code discussions
- Drive engagement and discovery
- Unique feature, hard to copy

### 4. **Build Community-Driven Content**
- Wiki-mode collaborative editing (Phase 1.5)
- Contributors improve content over time
- Community owns knowledge base
- Self-sustaining quality

### 5. **Go Freemium Eventually, Not Now**
- Launch free (Phase 0)
- Monetize in Phase 1.5 (Pro tier + Enterprise)
- Focus on growth first, revenue second
- Creator revenue share to incentivize quality

---

## The North Star

**Weekly Active Workflow Consumers:** Users consuming at least one workflow/skill/trend per week.

**Why:** Measures core engagement, includes both creators and consumers, indicates product-market fit.

**Targets:**
- Week 4: 500 WAU
- Week 8: 1,500 WAU (MVP complete)
- Month 6: 10,000 WAU
- Month 12: 50,000+ WAU

**If you miss:** Be ready to pivot. This is the signal.

## The MVP (8 Weeks)

### Features (11 total)
1. User auth & profiles
2. Article/workflow publishing
3. Skill registry (basic)
4. Trending X API feed
5. Search & discovery
6. Comments & voting
7. Multi-language support (ko/en/ja)
8. Upvotes & social feedback
9. Homepage & navigation
10. User dashboard
11. Community moderation basics

### What NOT to include
- Skill verification challenges (Phase 1.5)
- Wiki-mode editing (Phase 1.5)
- Learning paths (Phase 1.5)
- Team workspaces (Phase 1.5)
- Monetization (Phase 1.5)
- IDE extensions (Phase 2)
- AI recommendations (Phase 2)

### Success Criteria
- 1,000 registered users
- 100+ published articles
- 50+ active contributors
- 40%+ day-7 retention
- NPS 40+
- Zero critical bugs
