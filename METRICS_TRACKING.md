### Virality Coefficient

**Definition:** How many new users does each existing user bring?

**Calculation:**
- Week N new users / Week N-1 total users = coefficient
- Target: 1.2 (each user brings 0.2 new users)
- Tracking: Via referral link, shared articles, Twitter mentions

## Health Checks (Monthly)

### Product Health Score

Calculate monthly using formula:

```
Health Score = (Retention × 0.3) + (Engagement × 0.3) + (Growth × 0.2) + (Quality × 0.2)

Where:
- Retention = D7 retention % / 60% (target)
- Engagement = DAU / WAU ratio / 0.25 (target)
- Growth = MoM growth % / 50% (target)
- Quality = Avg rating / 4.5 (target)
```

**Interpretation:**
- 0.8+: Green (all systems healthy)
- 0.6-0.8: Yellow (monitor closely, address issues)
- < 0.6: Red (pivot needed, emergency mode)

## Red Flags (Immediate Action Triggers)

| Red Flag | Action |
|----------|--------|
| D7 retention drops below 30% | Emergency UX audit |
| Content publish rate < 1/day | Reach out to contributors personally |
| % search views declining | SEO/search UX issue (audit) |
| Contributor activation < 3% | Onboarding broken (test it) |
| NPS trending downward | User interview sprint |
| Trending CTR < 3% | Feature/positioning issue |
| Churn rate > 20% | User interview to find pain points |

---

## How to Use This Dashboard

1. **Weekly:** Update engagement metrics every Monday morning
2. **As things happen:** Log content, contributors, skill verifications daily
3. **End of month:** Calculate Health Score and review trends
4. **Quarterly:** Reset targets based on learnings

**Tools to Consider:**
- Mixpanel or Amplitude for event tracking
- Google Analytics 4 for website metrics
- Supabase/PostgreSQL for custom dashboards
- Tableau or Metabase for visualization
- Notion for manual tracking (during MVP)
